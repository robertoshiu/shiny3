// SOL.02 · SOLUTION FRAMEWORK

六層架構，六個解決框架

每一層對應一個清晰的業務問題域。理解分層，才能準確評估範圍、鎖定交付責任、衡量結果。

02
L1

輸入層 / INPUT

問題：異質機台協議分歧，訊號孤立，無法統一進入數據管道。

範圍：8 大工藝機台 SECS-GEM / GEM300 接入；OPC UA 廠務；EAP 設備自動化；邊緣預處理節點。

協議統一輸出
L2

數據層 / DATA

問題：高頻設備波形數據量大、時序複雜，傳統資料庫無法支撐 AI 特徵工程與 EDA 分析。

範圍：Historian 時序歸檔；Lakehouse EDA 高頻採集；NVMe 數據湖；數據治理與存取控制。

NVMe 結構化數據湖
L3

算力層 / COMPUTE

問題：AI 訓練與即時推理需要超算級算力，普通伺服器集群無法支撐 Digital Twin 與 Agentic RAG 同時運行。

範圍：3 × GB300 NVL72（液冷）AI Fabric；4 × HGX B300 邊緣推理；Quantum-X800 800Gb/s XDR 全互聯；Spectrum-X SN5600 東西向網。

AI Fabric 決策算力
L4

應用層 / APPLICATION

問題：製程知識分散在人腦與 Excel，機台異常後靠工程師人工研判，決策閉環耗時長且無法規模化。

範圍：高量產 MES 平台（FAB300）MES 核心；Agentic RAG / LLM 自主推理；NVIDIA Omniverse Digital Twin；ECR / ECO 工程變更管理。

HVM MES 自動決策
L5

治理層 / GOVERNANCE

問題：OT 與 IT 網絡邊界模糊，合規要求多軌並行，APT 攻擊與內部威脅難以統一可視化與響應。

範圍：CCC 中控台全廠事件整合；SOC 24×7 SIEM/SOAR 驅動；OT 入侵偵測與資產可視化；IEC 62443 / SEMI E187 / 等保 2.0 三級合規底座。

IEC 62443 雙域合規
L6

備援層 / RESILIENCE

問題：晶圓廠停機成本極高，單點故障或區域性災難可能造成不可逆產線損失，業務連續性需要架構級保障而非事後補救。

範圍：溫備援跨區域架構；異步複製與 SD-WAN 雙 ISP；2 × HGX B300 關鍵推理備援；DR 季度演練納入治理閉環；RTO ≤ 4hr / RPO ≤ 15min。

RTO ≤ 4hr 業務連續

AI 算力與儲存

AI COMPUTE & STORAGE
AI 算力佔比

// 解決的問題

安全佔比

// 解決的問題

異地備援 DR

DISASTER RECOVERY
備援架構

// 解決的問題

晶圓廠單次非計劃停機（設備故障、電力中斷、區域性災難）的損失金額極高 — 若無跨區域溫備援架構，恢復時間往往超過業務承受上限。事後備份策略在 AI 工作負載場景下無效，需要架構級的 RTO/RPO 保障。

企業整合

ENTERPRISE INTEGRATION
企業整合佔比

// 解決的問題

晶圓廠的製造執行層（ISA-95 L3）與企業管理層（L4：ERP、PLM）長期割裂 — 工單信息無法即時下沉至設備、良率數據無法自動上拋至財務系統，手工對賬與離線報表造成決策滯後，並引入數據一致性風險。

// 交付範圍

  • ISA-95 L3↔L4 數據流架構設計與實施，對齊 MES FAB300 接口規範
  • ERP / PLM 雙向接口（工單、BOM、庫存、良率報表）
  • AMHS / MCS 自動物料搬運系統整合，LIMS 實驗室信息管理接口
  • FMCS 廠務管控系統 OPC UA 接口，統一進入數據層

為何委外，而非自建？

現狀競爭者是在地自建 — 由晶圓廠自己的工程團隊整合 IT/OT+AI 全棧。這條路在 NVIDIA 世代 IT/OT 場景下面臨三道結構性難題。

05

人才根本招不到

TALENT SCARCITY

NVIDIA Blackwell 世代的 IT/OT+AI 整合人才極度稀缺 — 懂 SECS-GEM 設備接入、又能整合 GB300 NVL72 液冷 AI Fabric、再對齊 IEC 62443 合規底座的複合型工程師，全球市場上供給嚴重不足。即使找到，組建一支可完整交付的團隊需要數年，而您的量產爬坡時程不等人。
